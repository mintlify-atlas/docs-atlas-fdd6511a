## Prerequisites

Before you begin, ensure you have the following installed on your system:

<Note>
  **Required Software**
  - Node.js 20.x or higher
  - PostgreSQL 12 or higher
  - npm, yarn, or pnpm package manager
</Note>

<Steps>
  <Step title="Install Node.js">
    Download and install Node.js from [nodejs.org](https://nodejs.org/). Verify installation:
    ```bash
    node --version
    npm --version
    ```
  </Step>
  
  <Step title="Install PostgreSQL">
    Install PostgreSQL for your operating system:
    - **macOS**: `brew install postgresql@16`
    - **Ubuntu/Debian**: `sudo apt install postgresql postgresql-contrib`
    - **Windows**: Download from [postgresql.org](https://www.postgresql.org/download/windows/)
    
    Start the PostgreSQL service and verify:
    ```bash
    psql --version
    ```
  </Step>
  
  <Step title="Clone the Repository">
    Clone the GAOTEV repository to your local machine:
    ```bash
    git clone <repository-url>
    cd sistema-gestion
    ```
  </Step>
</Steps>

The GAOTEV project follows the Next.js 16 App Router structure:

```

├── app/
│   ├── api/

```

## Database Setup

<Steps>
  <Step title="Create Database">
    Create a PostgreSQL database for the project:
    ```bash
    createdb gaotev_dev
    ```
    
    Or using psql:
    ```sql
    CREATE DATABASE gaotev_dev;
    ```
  </Step>
  
  <Step title="Configure Environment Variables">
    Create a `.env` file in the project root (see [Configuration](/development/configuration) for details):
    ```bash
    DATABASE_URL="postgresql://username:password@localhost:5432/gaotev_dev"
    DIRECT_URL="postgresql://username:password@localhost:5432/gaotev_dev"
    NEXTAUTH_SECRET="your-secret-key"
    NEXTAUTH_URL="http://localhost:3000"
    ```
  </Step>
  
  <Step title="Run Migrations">
    Apply the database schema using Prisma:
    ```bash
    npx prisma migrate dev
    ```
    
    This will create the necessary tables based on the schema defined in `prisma/schema.prisma`.
  </Step>
  
  <Step title="Generate Prisma Client">
    Generate the Prisma Client to interact with the database:
    ```bash
    npx prisma generate
    ```
  </Step>
</Steps>

<Warning>
  Make sure your PostgreSQL server is running before executing migrations. The connection string format must include both `DATABASE_URL` and `DIRECT_URL` for proper Prisma connection pooling support.
</Warning>

### Development Features

<Accordion title="Hot Reloading">
  Next.js automatically reloads the page when you make changes to:
  - React components in the `app/` directory
  - API routes in `app/api/`
  - Styles in CSS files
  - Configuration files (requires server restart for some changes)

  - View your database with Prisma Studio: `npx prisma studio`
  - Create migrations: `npx prisma migrate dev --name migration_name`
  - Reset database: `npx prisma migrate reset`
  - Seed data (if seed script configured): `npx prisma db seed`
</Accordion>

<Accordion title="TypeScript Development">

  - Type checking happens automatically during development
  - Use `@/*` path alias to import from root (e.g., `import { prisma } from '@/lib/prisma'`)

</Accordion>

## Authentication Flow

The project uses NextAuth.js with credential-based authentication:

1. **Login Route**: `app/login/page.jsx` - User login interface
2. **Auth API**: `app/api/auth/[...nextauth]/route.ts` - Handles authentication
3. **Middleware**: `middleware.ts` - Protects `/dashboard` route
4. **Session**: Uses JWT strategy for session management

<Note>
  Protected routes are defined in `middleware.ts`. Currently, only `/dashboard` requires authentication.
</Note>

## Troubleshooting

### Common Issues

<Accordion title="Database Connection Errors">
  **Problem**: Cannot connect to PostgreSQL
  
  **Solutions**:
  - Verify PostgreSQL is running: `pg_isready`
  - Check connection string format in `.env`
  - Ensure database exists: `psql -l`
  - Verify credentials and permissions
</Accordion>

<Accordion title="Prisma Client Issues">
  **Problem**: Prisma Client not found or outdated
  
  **Solutions**:
  - Regenerate client: `npx prisma generate`
  - Clear node_modules: `rm -rf node_modules && npm install`
  - Check Prisma version compatibility
</Accordion>

<Accordion title="Port Already in Use">
  **Problem**: Port 3000 is already in use
  
  **Solutions**:
  - Kill process on port 3000: `lsof -ti:3000 | xargs kill -9`
  - Or use a different port: `PORT=3001 npm run dev`
</Accordion>

<Accordion title="NextAuth Secret Not Set">
  **Problem**: NextAuth requires NEXTAUTH_SECRET
  
  **Solutions**:
  - Generate a secret: `openssl rand -base64 32`
  - Add to `.env`: `NEXTAUTH_SECRET="generated-secret"`
  - Restart the development server
</Accordion>
