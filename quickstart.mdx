## Installation Steps

<Steps>
  <Step title="Clone the repository">
    Clone the GAOTEV repository to your local machine:
    
    ```bash
    git clone https://github.com/Gusivelin/GAOTEV.git
    cd GAOTEV
    ```
  </Step>
  
  <Step title="Install dependencies">
    Install all required npm packages. GAOTEV uses the following main dependencies:
    
    - **Next.js 16.1.6** - React framework
    - **NextAuth.js 4.24.13** - Authentication
    - **Prisma 6.19.2** - ORM and database toolkit
    - **React 19.2.3** - UI library
    - **Tailwind CSS 4** - Styling
    - **bcrypt 6.0.0** - Password hashing
    
    <CodeGroup>
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    
    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Configure environment variables">
    Create a `.env` file in the root directory with your database and authentication settings:
    
    ```bash .env
    # Database connection URLs
    DATABASE_URL="postgresql://user:password@localhost:5432/gaotev"
    DIRECT_URL="postgresql://user:password@localhost:5432/gaotev"
    
    # NextAuth.js configuration
    NEXTAUTH_SECRET="your-secret-key-here"
    NEXTAUTH_URL="http://localhost:3000"
    ```
    
    <Warning>
    Generate a secure `NEXTAUTH_SECRET` using:
    ```bash
    openssl rand -base64 32
    ```
    Never commit your `.env` file to version control.
    </Warning>
  </Step>
  
  <Step title="Set up PostgreSQL database">
    Create a new PostgreSQL database for GAOTEV:
    
    ```bash
    # Connect to PostgreSQL
    psql -U postgres
    
    # Create database
    CREATE DATABASE gaotev;
    
    # Exit psql
    \q
    ```
    
    Make sure your `DATABASE_URL` in `.env` matches your PostgreSQL credentials.
  </Step>
  
  <Step title="Run database migrations">
    Set up your database schema using Prisma migrations:
    
    ```bash
    npx prisma migrate dev
    ```
    
    This command will:
    - Read the schema from `prisma/schema.prisma`
    - Create the `User` table with the following structure:
    
    ```prisma schema.prisma
    model User {
      id        String   @id @default(cuid())
      email     String   @unique
      password  String
      createdAt DateTime @default(now())
    }
    ```
    
    - Apply the migration to your database
    - Generate the Prisma Client for type-safe queries
  </Step>
  
  <Step title="Seed a test user (Optional)">
    To create a test user, you can use Prisma Studio or create a seed script:
    
    ```bash
    npx prisma studio
    ```
    
    Or create a seed script `prisma/seed.ts`:
    
    ```typescript seed.ts
    import { PrismaClient } from '@prisma/client'
    import bcrypt from 'bcrypt'
    
    const prisma = new PrismaClient()
    
    async function main() {
      const hashedPassword = await bcrypt.hash('demo123', 10)
      
      const user = await prisma.user.create({
        data: {
          email: 'admin@demo.com',
          password: hashedPassword,
        },
      })
      
      console.log('Created test user:', user.email)
    }
    
    main()
      .catch(console.error)
      .finally(() => prisma.$disconnect())
    ```
    
    Run the seed:
    ```bash
    npx tsx prisma/seed.ts
    ```
  </Step>
  
  <Step title="Start the development server">
    Launch the application in development mode:
    
    ```bash
    npm run dev
    ```
    
    The application will start on `http://localhost:3000`. You should see output like:
    
    ```bash
    ▲ Next.js 16.1.6
    - Local:        http://localhost:3000
    - Network:      http://192.168.1.x:3000
    
    ✓ Ready in 2.1s
    ```
  </Step>
  
  <Step title="Access the application">
    Open your browser and navigate to `http://localhost:3000`. You'll be automatically redirected to the login page.
    
    If you created a test user in Step 6, you can log in with:
    - **Email**: `admin@demo.com`
    - **Password**: `demo123`
  </Step>
</Steps>

## Logging In

The login page uses NextAuth.js credentials provider for authentication. Here's how the sign-in flow works:

```jsx page.jsx
const handleSubmit = async (e) => {
  e.preventDefault()
  setError("")

  const result = await signIn("credentials", {
    email,
    password,
    redirect: false,
  })

  if (result?.ok) {
    router.push("/dashboard")
  } else {
    setError("Credenciales incorrectas")
  }
}
```

After successful authentication, you'll be redirected to the dashboard at `/dashboard`.

## Route Protection

GAOTEV uses Next.js middleware to protect routes. The middleware checks for a valid session before allowing access:

```typescript middleware.ts
export { default } from "next-auth/middleware"

export const config = {
  matcher: ["/dashboard"],
}
```

If you try to access `/dashboard` without being authenticated, you'll be automatically redirected to the sign-in page.

## Verification

To verify your installation is working correctly:

<AccordionGroup>
  <Accordion title="Check the login page">
    Visit `http://localhost:3000` and confirm you see the login form with email and password fields.
  </Accordion>
  
  <Accordion title="Test authentication">
    Try logging in with invalid credentials - you should see an error message. Then try with valid credentials - you should be redirected to the dashboard.
  </Accordion>
  
  <Accordion title="Check database connection">
    Run `npx prisma studio` to open Prisma Studio and verify your database schema is correct and data is present.
  </Accordion>
  
  <Accordion title="Verify session management">
    After logging in, open browser DevTools → Application → Cookies and check for the NextAuth session token.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion icon="database" title="Database connection errors">
    If you see database connection errors:
    
    - Verify PostgreSQL is running: `pg_isready`
    - Check your `DATABASE_URL` format and credentials
    - Ensure the database exists: `psql -l | grep gaotev`
    - Try connecting manually: `psql postgresql://user:password@localhost:5432/gaotev`
  </Accordion>
  
  <Accordion icon="key" title="NextAuth errors">
    If authentication isn't working:
    
    - Verify `NEXTAUTH_SECRET` is set in `.env`
    - Check `NEXTAUTH_URL` matches your development URL
    - Clear browser cookies and try again
    - Check browser console for JavaScript errors
  </Accordion>
  
  <Accordion icon="circle-exclamation" title="Port already in use">
    If port 3000 is already in use:
    
    ```bash
    # Find process using port 3000
    lsof -ti:3000
    
    # Kill the process (macOS/Linux)
    kill -9 $(lsof -ti:3000)
    
    # Or use a different port
    PORT=3001 npm run dev
    ```
  </Accordion>
  
  <Accordion icon="cube" title="Prisma Client errors">
    If you see Prisma Client errors:
    
    - Regenerate the client: `npx prisma generate`
    - Run migrations again: `npx prisma migrate dev`
    - Reset the database if needed: `npx prisma migrate reset`
  </Accordion>
</AccordionGroup>
